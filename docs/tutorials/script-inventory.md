# Script Inventory Demo

ManifestMark can summarize the package scripts a reviewer should run before a
release or handoff. This demo turns the checked-in fixtures into small Markdown
snippets for build and test tasks.

## Run it

```sh
bash demo/run-script-inventory.sh
```

The script writes:

- `.manifestmark-script-inventory/single-build.md`
- `.manifestmark-script-inventory/single-test.md`
- `.manifestmark-script-inventory/workspace-test.md`

Each file is generated by the `manifestmark scripts` command and verified with a
targeted text check so the demo fails if the fixture output changes
unexpectedly.

## Review workflow

Use the script inventory after `manifestmark scan` has checked the package
metadata. It gives a reviewer a quick view of the exact build or test command
declared by the manifest without running that command.

```sh
manifestmark scripts . --task test
```

For workspaces, the output preserves the root command such as `pnpm -r test`, so
the handoff stays grounded in the manifest rather than a guessed convention.
